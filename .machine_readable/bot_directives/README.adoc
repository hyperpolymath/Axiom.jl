= Bot directives — Axiom.jl
:toc:

== Purpose

Per-repo directives for automated agents operating on
`hyperpolymath/Axiom.jl`. These files tell bots what this repository
considers safe, what is forbidden, and which findings are already
adjudicated, so automated runs do not relitigate settled decisions or
touch protected surfaces.

== Precedence

. Maintainer instruction (see `MAINTAINERS.adoc`) — always wins.
. These directives.
. Bot built-in defaults.

== Files

Estate-standard trio (net-new 2026-06-13):

* `hypatia.a2ml` — Hypatia scanner config pointers + pre-adjudicated
  finding classes (Zig/Idris2 ABI ccall boundaries, ENV-in-test pattern),
  grounded in the repo's test suite and architecture.
* `gitbot-fleet.a2ml` — fleet roster, branch policy, never-touch paths,
  per-bot constraints.
* `git-private-farm.a2ml` — `.git-private-farm` propagation contract;
  propagation is **live** here (the repo ships
  `.github/workflows/instant-sync.yml`).

Agent-methodology files (pre-existing, renamed from agent_instructions/):

* `methodology.a2ml` — default agent mode, invariants, ring ceiling, priority weights.
* `coverage.a2ml` — session coverage tracking.
* `debt.a2ml` — meander debt list.

== Repo-specific ground rules (summary)

* Axiom.jl is a Julia ML framework with a Zig compute layer, Idris2 ABI,
  and formal verification surface — these are bot-off-limits without review.
* Protected surfaces: `src/verification/`, `src/Abi/`, `zig/`,
  `packages/SMTLib.jl/`, `.claude/CLAUDE.md`, `.github/workflows/`.
* Propagation to `.git-private-farm` is live via `instant-sync.yml`;
  secret used is `FARM_DISPATCH_TOKEN` (name only).
* CI green precedes merge; bots never auto-merge and never delete
  branches. Escalation channel is an issue, not PR-comment spam.
